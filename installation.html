The stable version of oce is provided on the R archives. It is installed from within R with

{% highlight R %} install.packages("gsw") {% endhighlight %}

Users who are set up to build packages (which requires a C compiler, etc) may stay at the cutting edge by installing a development version. This may be done in several ways, of which the third is probably the best for most users.

  1. Download the source for oce and ocedata as tarballs (oce and ocedata), expand the files, rename the resultant directories to oce and ocedata and then execute the OS commands stated below. {% highlight bash %} R CMD build oce R CMD install oce_0.9-15.tar.gz # adjust version as needed R CMD build ocedata R CMD install ocedata_0.1.1.gz # adjust version as needed {% endhighlight %} A similar procedure works for the zip files at oce and ocedata.
  2. Acquire the sources through Git using {% highlight bash %} git clone https://dankelley@github.com/dankelley/oce.git git clone https://dankelley@github.com/dankelley/ocedata.git {% endhighlight %} and then build them as described in #1.
  3. Download and build using install_github() {% highlight R %} library(devtools) install_github("oce", "dankelley", "develop") install_github("ocedata", "dankelley", "master") {% endhighlight %}

Some Windows users find it hard to install from source, so they may visit dropbox.com to get binary (zip) files. Since these files tend to be somewhat out of date, the best plan is to get set up to build packages.
